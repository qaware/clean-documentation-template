= This is a bad example

This is a bad example of AsciiDoc as it doesn't follow the "one line per sentence" rule. If you just write each sentence after each other the diff's and therefore merges are getting worse to handle. Markdown and AsciiDoc will handle single new lines as blanks. In the rendered output no difference can be spotted. If you add a https://github.com/qaware[link] then it gets even worse.

Run this example with `git diff --no-index one-line-per-sentence.bad-1.adoc one-line-per-sentence.bad-2.adoc`.
